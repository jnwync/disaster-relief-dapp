# ChainRelief Sprint Workspace (5-Day)

This folder contains owner-specific execution sheets derived from `docs/sprint-plan.md`.

## Files

- `aaron.md` - contract deployment, integration, contract bug fixes
- `allan.md` - QA execution, matrix completion, error-mapping validation
- `wayne.md` - frontend coordination, env sync, frontend bug fixes
- `rhea.md` - responsive, polish, accessibility, footer
- `chito.md` - docs, demo script, about page, submission package

## Daily Standup Format (2-3 minutes per person)

1. What was completed yesterday (proof: PR, commit, screenshot, or test result)
2. What will be completed today
3. Blockers and owner needed

## Handoff Rules

1. Owner marks item complete in their file.
2. Owner posts proof link in group chat.
3. Receiver acknowledges handoff before starting dependent work.

## Critical Path (must not slip)

1. W1 deployment on Sepolia
2. W2 frontend env update with Sepolia config
3. W3 full integration test
4. W4-W6 QA pass and bug fixing loop
5. W19 rehearsal + W20 backup video by Day 5
